󰏚 기술 구현 내용

❍ 시스템 아키텍처 개요
   - 통합 풀스택 구조: Next.js 16(App Router)을 기반으로, API Router를 백엔드 API 레이어이자 데이터베이스 단일 작성 지점(sole writer)으로 삼아 클라이언트·서버·AI 워커 간 책임을 분리하여 구현함.
   - 데이터·스토리지: Supabase(PostgreSQL + Object Storage + Auth)를 채택하여, 캐릭터·생성 이력·크레딧 등 정형 데이터는 PostgreSQL에, 레퍼런스·결과 이미지는 Object Storage에 저장하고 서명 URL(signed URL)로 접근을 제어함.
   - 비동기 AI 실행 계층: 수초~수분이 소요되는 이미지 생성을 HTTP 요청 경로에서 분리하여, 큐 기반 외부 워커(Trigger.dev)가 비동기로 처리하도록 구현함. 큐·재시도·멱등성으로 생성 작업의 신뢰성을 확보함.
   - AI 모델 추상화: Google Gemini Image, OpenAI 이미지 생성 모델을 어댑터(adapter) 패턴으로 추상화하여, 모델 교체·병행 및 향후 모델 확장이 용이하도록 구현함.
   - 실시간 3D 포즈 에디터: Three.js(react-three-fiber) 기반 3D 인체 마네킹을 실시간 조작하고, 그 결과를 2D 이미지로 캡처하여 AI 생성 입력으로 변환함.

❍ 핵심 기능별 기술 구현

   ① 컷 설계 기능
      - 컷 레이아웃을 정형 데이터(JSON 스키마)로 모델링하여 패널의 위치·크기·순서 및 장면 흐름을 저장·재현하도록 설계함.
      - 캔버스 기반 레이아웃 에디터(드래그·리사이즈)로 컷을 구성하고, 각 컷에 포즈·카메라 시점·캐릭터 설정값(②~④)을 연결하여 컷 단위로 생성 파라미터를 관리하도록 구현함.
      - 컷 단위 메타데이터를 AI 생성 파이프라인에 전달하여 장면 간 연출·구도 일관성을 유지함.

   ② 포즈 프리셋 기능
      - Three.js 기반 3D 인체 마네킹(스킨드 메시 + 관절 본 계층)을 실시간 조작하며, 본별 회전값을 인체 가동범위(range of motion) 한도 내로 자동 보정하여 비현실적 포즈를 방지하도록 구현함.
      - 자주 쓰이는 포즈를 프리셋(기본 자세·앉기·액션 등)으로 제공하고, 사용자가 직접 만든 포즈를 저장·재사용할 수 있도록 구현함.
      - 조작 결과를 WebGL 렌더 캡처로 2D 포즈 레퍼런스 이미지로 변환하여, 관절 각도·자세 정보를 AI 모델이 해석 가능한 이미지 입력값으로 전달함.

   ③ 카메라 시점 조정 기능
      - 3D 씬 카메라(위치·타깃·화각(FOV))를 직접 제어하고, 정면·측면·하이앵글·로우앵글 등 시점 프리셋을 카메라 파라미터로 매핑하여 원클릭으로 장면 구도를 설정하도록 구현함.
      - 선택된 시점은 포즈 캡처 이미지의 구도에 반영됨과 동시에 텍스트 프롬프트의 구도 지시로 전달되어, 생성된 컷의 앵글·구도 일관성을 확보함.

   ④ 캐릭터 설정 저장 기능
      - 캐릭터 메타데이터 스키마(이름·메인 컨셉·성별·신체 비율(등신)·체형·스타일 노트·키워드 태그·레퍼런스 이미지)를 정의하여 PostgreSQL에 저장하고, 레퍼런스 이미지는 Object Storage에 보관하도록 구현함.
      - 저장된 설정값은 이후 모든 이미지 생성 과정에서 반복 활용되어 캐릭터의 외형·의상·분위기 등 정체성 일관성을 유지하며, 사용자별 소유권 기반 접근제어(Row Level Security)로 데이터를 격리함.

   ⑤ AI 이미지 생성 기능
      - 포즈 캡처 이미지(②), 카메라 시점(③), 캐릭터 레퍼런스·메타데이터(④)를 구조화된 프롬프트로 조립하고, 레퍼런스·포즈 이미지를 다중 이미지 조건(multi-image conditioning)으로 AI 생성 API에 전달하여 웹툰 컷 후보 이미지를 생성하도록 구현함.
      - 생성 요청은 큐에 적재되어 비동기로 처리되며(상태기계: 대기→처리중→완료/실패), 결과는 서명 검증된 콜백으로 회수하여 데이터베이스에 기록하고 사용자에게 결과 후보를 제공함.
      - 멱등성(idempotency)·자동 재시도·타임아웃 회수 메커니즘으로 생성 작업의 신뢰성을 확보하고, 프롬프트 엔지니어링(이미지 역할 명시·정체성 보존·3D 마네킹 아티팩트 억제)으로 컷 후보 품질을 향상함.

❍ 공통 기반 기술
   - 인증·계정: Supabase Auth(Google OAuth) 기반 로그인과 세션 관리, 사용자별 데이터 소유권 격리를 구현함.
   - 크레딧·결제: 토스페이먼츠 결제창(SDK·API) 연동과 서버 크레딧 원장(ledger)을 구현하여, 충전·생성 시 예약 차감 및 실패 시 자동 환불을 멱등적으로 처리함.
   - 신뢰성·보안: 트랜잭션 아웃박스(transactional outbox)·멱등 키·HMAC 서명 콜백·미처리 작업 회수(reaper)로 비동기 환경의 정확성을 보장하고, 결제 금액 서버 검증·데이터베이스 함수 권한 최소화·업로드 검증 등 보안 통제를 적용함.
